Progettata per funzionare senza interruzioni.

DigiByte è una blockchain UTXO multi-algoritmo, altamente sicura e pensata per scalare. Con tempi di blocco di 15 secondi e dodici anni di operatività ininterrotta, la rete offre una base testata sul campo per DGB, DigiAssets, l'autenticazione decentralizzata Digi-ID e — molto presto — DigiDollar.

Perché DigiByte

Progettata per il prossimo decennio del cripto.

Una catena UTXO in pura proof-of-work che dal 2014 introduce delle prime assolute — DigiShield, MultiAlgo, SegWit, Odocrypt, Dandelion++, DigiAssets e Digi-ID.

15s
Tempo di blocco

Blocchi da 15 s, regolati in fretta.

40× più veloce di Bitcoin e 10× più veloce di Litecoin. Le conferme in quindici secondi fanno sì che pagamenti POS, depositi su exchange e trasferimenti di asset si finalizzino nel tempo di leggere questa frase — senza la congestione o i picchi di commissioni delle catene più lente.

5
Consenso

Cinque algoritmi di mining in parallelo.

DigiByte protegge i blocchi con cinque algoritmi di proof-of-work indipendenti — SHA-256d, Scrypt, Skein, Qubit e Odocrypt. Ognuno si rivolge a un ecosistema di mining diverso (CPU, GPU, FPGA, ASIC) e Odocrypt si ri-aleatorizza ogni 10 giorni per impedire la dominanza degli ASIC. Attaccare la catena significa attaccare cinque algoritmi insieme.

21B
Offerta

Cap fisso, scarsa per design.

DGB è limitata matematicamente a 21 miliardi di monete, emesse secondo un calendario trasparente di riduzione mensile. Niente dev tax, niente sblocchi di governance, niente allocazioni nascoste. La scarsità prevedibile è ciò che rende credibile un asset monetario — ed è perché ogni byte di DGB porta valore dimostrabile e verificabile nel tempo.

12 anni live
Track record

Dodici anni live. Zero downtime.

Da gennaio 2014 la mainnet produce blocchi ogni quindici secondi senza interruzioni. Niente re-org profonde, niente fork di governance, niente hard fork di emergenza. Questo uptime, insieme a migliaia di nodi operati in modo indipendente nel mondo, è il tipo di track record operativo che serve all'infrastruttura critica — e che quasi nessun'altra catena può mostrare.

Digi-ID

Autenticazione open source, senza password.

Digi-ID si basa su firme crittografiche decentralizzate al posto delle password tradizionali, proteggendo l'identità dell'utente senza intermediari centralizzati.

  • Storage locale delle chiavi per la massima privacy.
  • Design nativamente resistente al phishing.
  • Integrazione drop-in per accessi digitali e fisici.
Apri Digi-ID.io Librerie
In cosa è diverso da OAuth o «Accedi con X»?

Il single sign-on tradizionale dipende da intermediari di dati centralizzati. Digi-ID è completamente decentralizzato: l'utente si autentica firmando una sfida unica direttamente dal proprio dispositivo, eliminando del tutto i server di terze parti dal processo.

Serve un wallet di criptovalute?

No, possedere criptovalute è opzionale. Molti wallet DigiByte supportano nativamente Digi-ID, ma puoi anche usare un'app di autenticazione standalone pensata solo per login sicuri.

Come la integro?

Gli sviluppatori possono usare le nostre librerie drop-in per Node.js, PHP, Python e Go oppure installare il plugin WordPress nativo. Tutte le risorse sono nel developer hub qui sotto.

Developer hub

Infrastruttura senza compromessi per lo sviluppo UTXO.

Accedi a una RPC affidabile e testata sul campo, con commissioni fisse e prevedibili. DigiByte fornisce la base per mettere in piedi nodi in pochi minuti e costruire su una rete davvero decentralizzata.

~/digibyte — bash
# Get current chain info $ digibyte-cli getblockchaininfo { "chain": "main", "blocks": 21000000, "difficulty": 128456.7, "verificationprogress": 0.99999 } 
// Send 100 DGB via Core RPC (Node.js, no external deps)
import http from 'node:http';
const body = JSON.stringify({ jsonrpc: '1.0', id: 'web', method: 'sendtoaddress',
  params: ['DXXXXXXXXXXXXXXXXXXXXXXXXXXXX', 100] });

const req = http.request({
  hostname: '127.0.0.1', port: 14022, method: 'POST',
  auth: 'rpcuser:rpcpass',
  headers: { 'Content-Type': 'application/json', 'Content-Length': body.length }
}, res => {
  let d = ''; res.on('data', c => d += c);
  res.on('end', () => console.log(JSON.parse(d)));
});
req.write(body); req.end();
# Listen for new blocks (Python, requests)
import requests, time

URL = "http://rpcuser:rpcpass@127.0.0.1:14022"
last = 0
while True:
    info = requests.post(URL, json={
        "jsonrpc": "1.0", "id": "py", "method": "getblockchaininfo", "params": []
    }).json()["result"]
    if info["blocks"] != last:
        print("new block:", info["blocks"])
        last = info["blocks"]
    time.sleep(5)
curl -u rpcuser:rpcpass --data-binary \
  '{"jsonrpc":"1.0","id":"sh","method":"getmempoolinfo","params":[]}' \
  -H 'content-type: application/json;' http://127.0.0.1:14022/

Perché rilasciare su DGB?
  • Mercati di commissioni prevedibili: costi di transazione sotto il centesimo permettono di scalare l'app in modo sostenibile.
  • Transazioni senza approvazioni: la meccanica nativa UTXO evita i rischi di sicurezza legati ad ampie allowance sui token.
  • Esecuzione senza stato: un ambiente UTXO puro permette uno sviluppo semplice, parallelo e componibile.
Track record

Oltre un decennio di esecuzione continua.

Dall'attivazione precoce di SegWit al consenso multi-algoritmo. Ogni upgrade di protocollo è stato open source, testato in mainnet e rilasciato senza finanziamenti centralizzati.

  1. 2014

    Genesis

    Lancio equo via mining pubblico; 0,5% di premine distribuito tra giveaway e sviluppo.

  2. 2014

    DigiShield

    Ri-tarature della difficoltà in tempo reale per garantire tempi di blocco stabili.

  3. 2014

    MultiAlgo

    Cinque algoritmi di mining in parallelo per evitare la centralizzazione dell'hashrate.

  4. 2017

    SegWit

    Upgrade di protocollo che ottimizza la capacità e risolve la malleabilità delle transazioni.

  5. 2019

    Odocrypt & Dandelion++

    Consenso resistente agli ASIC e routing delle transazioni che preserva la privacy.

  6. 2020+

    DigiAssets & Digi-ID

    Integrazione a livello di ecosistema per asset on-chain e autenticazione decentralizzata.

  7. 2025

    Core v9.26.4

    Infrastruttura di nodo modernizzata, basata sul framework Bitcoin Core 26.

    • BIP340 — Schnorr signatures
    • BIP341 — Pay-to-Taproot (P2TR)
    • BIP342 — Tapscript
  8. 2026

    DigiDollar Live on mainnet

    Una stablecoin totalmente non-custodial che genera liquidità ancorata al dollaro tramite riserve DGB con time-lock.
