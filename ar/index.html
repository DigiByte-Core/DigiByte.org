لماذا DigiByte

مصمم لعقد العملات المشفّرة القادم.

سلسلة UTXO تعتمد على إثبات العمل النقي وقدّمت ابتكارات أوّلية في القطاع منذ 2014 — DigiShield، MultiAlgo، SegWit، Odocrypt، Dandelion++، DigiAssets وDigi-ID.

15s
زمن الكتلة

كتل 15 ثانية، تسوية سريعة.

أسرع 40× من Bitcoin و10× من Litecoin. التأكيدات في خمس عشرة ثانية تعني أن مدفوعات نقاط البيع وودائع منصات التداول وتحويلات الأصول تنتهي خلال الوقت الذي يستغرقه قراءة هذه الجملة — دون الازدحام أو ارتفاع الرسوم الذي تشهده السلاسل الأبطأ.

5
الإجماع

خمس خوارزميات تعدين بالتوازي.

يؤمّن DigiByte الكتل عبر خمس خوارزميات إثبات عمل مستقلة — SHA-256d, Scrypt, Skein, Qubit وOdocrypt. تستهدف كل واحدة منظومة تعدين مختلفة (CPU، GPU، FPGA، ASIC) ويعيد Odocrypt عشوائيًا توليد نفسه كل 10 أيام لمنع هيمنة ASIC. مهاجمة السلسلة تعني مهاجمة خمس خوارزميات في وقت واحد.

21B
العرض

سقف صارم، ندرة بالتصميم.

DGB محدود رياضيًا بـ 21 مليار عملة، تُصدر وفق جدول شفّاف يتناقص شهريًا. لا ضريبة للمطوّرين، ولا إطلاقات حوكمة، ولا تخصيصات خفية. الندرة القابلة للتنبؤ هي ما يجعل أصلًا نقديًا موثوقًا — ولذلك يحمل كل بايت من DGB قيمة قابلة للإثبات والتحقق عبر الزمن.

السجل

اثنا عشر عامًا من التشغيل. صفر تعطّل.

منذ يناير 2014، أنتجت الشبكة الرئيسية كتلة كل خمس عشرة ثانية دون انقطاع. لا re-orgs عميقة، ولا فورك حوكمة، ولا هارد فورك طارئ. هذا الجاهزية، مع آلاف العقد المُدارة باستقلال حول العالم، هي تمامًا نوع السجل التشغيلي الذي تحتاجه البنية التحتية الحيوية — ولا تستطيع أي سلسلة أخرى تقريبًا إظهاره.

DigiAssets

أصدر أصولًا مخصصة على سلسلة UTXO آمنة.

أصدر أصولًا رقمية مخصصة بشكل أصلي على شبكة DigiByte — من الرموز والعملات المستقرة إلى سجلات الهوية. تُسوّى كل أصل خلال 15 ثانية وتؤمَّن بإجماع إثبات عمل عالمي.

اختر أصلك

رمز قابل للاستبدال، مقتنيات، شهادة، قسيمة — أي شيء يمكن سكّه على الطبقة الأساسية لـ UTXO.

حدّد البيانات الوصفية

حدّد الاسم والعرض والقابلية للقسمة والبيانات الوصفية المثبتة على IPFS كي تعرضها المحافظ والمستكشفات بشكل أصلي.

Digi-ID

مصادقة مفتوحة المصدر دون كلمة مرور.

يعتمد Digi-ID على توقيعات تشفيرية لامركزية بدلًا من كلمات المرور التقليدية، ويحمي هوية المستخدم بلا وسطاء مركزيين.

  • تخزين محلي للمفاتيح لخصوصية مطلقة.
  • تصميم مقاوم بطبيعته للتصيد.
  • تكامل جاهز للاستخدام للوصول الرقمي والمادي.
افتح Digi-ID.io المكتبات
كيف يختلف عن OAuth أو «تسجيل الدخول بـ X»؟

يعتمد تسجيل الدخول الموحّد التقليدي على وسطاء بيانات مركزيين. Digi-ID لامركزي بالكامل: يصادق المستخدمون بتوقيع تحدٍّ فريد مباشرة من أجهزتهم، ما يزيل خوادم الطرف الثالث من العملية بالكامل.

هل يتطلب محفظة عملات مشفّرة؟

لا، امتلاك العملات المشفّرة اختياري. تدعم العديد من محافظ DigiByte Digi-ID أصليًا، لكن يمكنك أيضًا استخدام تطبيق مصادقة مستقل مصمم حصريًا لتسجيلات الدخول الآمنة.

كيف أدمجه؟

يستطيع المطوّرون استخدام مكتباتنا الجاهزة لـ Node.js وPHP وPython وGo، أو تثبيت إضافة WordPress الأصلية. تجد جميع الموارد في مركز المطوّرين أدناه.

مركز المطوّرين

بنية تحتية دون مساومات لتطوير UTXO.

اطّلع على RPC موثوق ومجرّب ميدانيًا بهياكل رسوم ثابتة وقابلة للتنبؤ. يوفّر DigiByte الأساس لنشر العقد خلال دقائق وللبناء على شبكة لامركزية فعلًا.

~/digibyte — bash
# Get current chain info $ digibyte-cli getblockchaininfo { "chain": "main", "blocks": 21000000, "difficulty": 128456.7, "verificationprogress": 0.99999 } 
// Send 100 DGB via Core RPC (Node.js, no external deps)
import http from 'node:http';
const body = JSON.stringify({ jsonrpc: '1.0', id: 'web', method: 'sendtoaddress',
  params: ['DXXXXXXXXXXXXXXXXXXXXXXXXXXXX', 100] });

const req = http.request({
  hostname: '127.0.0.1', port: 14022, method: 'POST',
  auth: 'rpcuser:rpcpass',
  headers: { 'Content-Type': 'application/json', 'Content-Length': body.length }
}, res => {
  let d = ''; res.on('data', c => d += c);
  res.on('end', () => console.log(JSON.parse(d)));
});
req.write(body); req.end();
# Listen for new blocks (Python, requests)
import requests, time

URL = "http://rpcuser:rpcpass@127.0.0.1:14022"
last = 0
while True:
    info = requests.post(URL, json={
        "jsonrpc": "1.0", "id": "py", "method": "getblockchaininfo", "params": []
    }).json()["result"]
    if info["blocks"] != last:
        print("new block:", info["blocks"])
        last = info["blocks"]
    time.sleep(5)

السجل

أكثر من عقد من التنفيذ المتواصل.

من تفعيل SegWit المبكر إلى الإجماع متعدد الخوارزميات. كل ترقية بروتوكولية كانت مفتوحة المصدر ومجرّبة على الشبكة الرئيسية ونُشرت دون تمويل مركزي.

  1. 2014

    Genesis

    إطلاق عادل عبر التعدين العام؛ premine 0.5% وزّعت على giveaways والتطوير.

  2. 2014

    DigiShield

    إعادة استهداف الصعوبة فوريًا لضمان أزمنة كتل ثابتة.

  3. 2014

    MultiAlgo

    خمس خوارزميات تعدين متوازية لمنع مركزة معدل التجزئة.

  4. 2017

    SegWit

    ترقية بروتوكولية تحسّن السعة وتصلح مرونة المعاملات.

  5. 2019

    Odocrypt & Dandelion++

    إجماع مقاوم لـ ASIC وتوجيه معاملات يحافظ على الخصوصية.

  6. 2020+

    DigiAssets & Digi-ID

    تكامل على مستوى المنظومة للأصول على السلسلة والمصادقة اللامركزية.

  7. 2025

    Core v8.26.2

    بنية تحتية محدّثة للعقد مبنية على إطار Bitcoin Core 26.

  8. 2026

    DigiDollar قريبًا

    عملة مستقرة غير حافظة بالكامل تولّد سيولة مرتبطة بالدولار الأمريكي عبر احتياطيات DGB مقفلة زمنيًا.
