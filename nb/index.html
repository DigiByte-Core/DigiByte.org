Konstruert for kontinuerlig drift.

DigiByte er en svært sikker UTXO-blokkjede med flere algoritmer, bygget for skala. Med blokker hvert 15. sekund og tolv år med uavbrutt oppetid leverer nettverket et kamptestet fundament for DGB, DigiAssets, desentralisert Digi-ID-autentisering og — veldig snart — DigiDollar.

Hvorfor DigiByte

Konstruert for kryptoens neste tiår.

En ren proof-of-work UTXO-kjede som har levert bransje-førsteheter siden 2014 — DigiShield, MultiAlgo, SegWit, Odocrypt, Dandelion++, DigiAssets og Digi-ID.

15s
Blokktid

15-sekunders blokker, raskt oppgjort.

40× raskere enn Bitcoin og 10× raskere enn Litecoin. Femten sekunders bekreftelser betyr at POS-betalinger, innskudd på børser og asset-overføringer fullføres på tiden det tar å lese denne setningen — uten tregere kjeders kø eller gebyrtopper.

5
Konsensus

Fem mining-algoritmer parallelt.

DigiByte sikrer blokker gjennom fem uavhengige proof-of-work-algoritmer — SHA-256d, Scrypt, Skein, Qubit og Odocrypt. Hver retter seg mot et eget mining-økosystem (CPU, GPU, FPGA, ASIC), og Odocrypt re-randomiserer seg selv hver 10. dag for å holde ASIC-dominans borte. Å angripe kjeden betyr å angripe fem algoritmer samtidig.

21B
Tilbud

Hardt tak, knapp by design.

DGB er matematisk begrenset til 21 milliarder mynter, utstedt etter en transparent månedlig reduksjonsplan. Ingen dev-skatt, ingen governance-opplåsinger, ingen skjulte allokeringer. Forutsigbar knapphet gjør et monetært aktiv troverdig — og det er derfor hver byte DGB bærer påvisbar, etterprøvbar verdi over tid.

12 år live
Track record

Tolv år live. Null nedetid.

Siden januar 2014 har mainnettet produsert blokker hvert femtende sekund uten avbrudd. Ingen dype re-orgs, ingen governance-forkser, ingen nød-hard forkser. Den oppetiden, sammen med tusenvis av uavhengig drevne noder verden over, er nettopp den slags driftshistorikk kritisk infrastruktur krever — og som nesten ingen annen kjede kan vise til.

Digi-ID

Passordfri, åpen kildekode-autentisering.

Digi-ID baserer seg på desentraliserte kryptografiske signaturer i stedet for tradisjonelle passord, og sikrer brukerens identitet uten sentrale mellomledd.

  • Lokal nøkkellagring for absolutt personvern.
  • Innebygd phishing-resistent design.
  • Drop-in-integrasjon for digital og fysisk tilgang.
Åpne Digi-ID.io Biblioteker
Hvordan skiller det seg fra OAuth eller «Logg inn med X»?

Tradisjonell single sign-on baserer seg på sentrale datameglere. Digi-ID er fullt desentralisert: brukere autentiserer ved å signere en unik utfordring direkte fra enheten sin, noe som fjerner tredjepartsservere helt fra prosessen.

Krever det en kryptolommebok?

Nei, det er valgfritt å eie krypto. Mange DigiByte-lommebøker støtter Digi-ID nativt, men du kan også bruke en frittstående autentiseringsapp som kun er laget for sikre innlogginger.

Hvordan integrerer jeg det?

Utviklere kan bruke våre drop-in-biblioteker for Node.js, PHP, Python og Go, eller installere den native WordPress-pluginen. Fullstendige ressurser finnes i utviklerhubben under.

Utviklerhub

Kompromissløs infrastruktur for UTXO-utvikling.

Få tilgang til en pålitelig, kamptestet RPC med faste, forutsigbare gebyrstrukturer. DigiByte gir grunnlaget for å rulle ut noder på minutter og bygge på et virkelig desentralisert nettverk.

~/digibyte — bash
# Get current chain info $ digibyte-cli getblockchaininfo { "chain": "main", "blocks": 21000000, "difficulty": 128456.7, "verificationprogress": 0.99999 } 
// Send 100 DGB via Core RPC (Node.js, no external deps)
import http from 'node:http';
const body = JSON.stringify({ jsonrpc: '1.0', id: 'web', method: 'sendtoaddress',
  params: ['DXXXXXXXXXXXXXXXXXXXXXXXXXXXX', 100] });

const req = http.request({
  hostname: '127.0.0.1', port: 14022, method: 'POST',
  auth: 'rpcuser:rpcpass',
  headers: { 'Content-Type': 'application/json', 'Content-Length': body.length }
}, res => {
  let d = ''; res.on('data', c => d += c);
  res.on('end', () => console.log(JSON.parse(d)));
});
req.write(body); req.end();
# Listen for new blocks (Python, requests)
import requests, time

URL = "http://rpcuser:rpcpass@127.0.0.1:14022"
last = 0
while True:
    info = requests.post(URL, json={
        "jsonrpc": "1.0", "id": "py", "method": "getblockchaininfo", "params": []
    }).json()["result"]
    if info["blocks"] != last:
        print("new block:", info["blocks"])
        last = info["blocks"]
    time.sleep(5)

Hvorfor levere på DGB?
  • Forutsigbare gebyrmarkeder: transaksjonskostnader under én cent gir bærekraftig app-skalering.
  • Approval-frie transaksjoner: den innebygde UTXO-mekanikken unngår sikkerhetsrisikoene ved brede token-tillatelser.
  • Stateless eksekvering: et rent UTXO-miljø gir enkel, parallell og komponerbar utvikling.

Track record

Over et tiår med kontinuerlig levering.

Fra tidlig SegWit-aktivering til multi-algoritme-konsensus. Hver protokolloppgradering har vært åpen kildekode, mainnet-testet og rullet ut uten sentral finansiering.

  1. 2014

    Genesis

    Rettferdig oppstart via offentlig mining; 0,5 % premine fordelt til giveaways og utvikling.

  2. 2014

    DigiShield

    Sanntidsjustering av vanskelighet for stabile blokktider.

  3. 2014

    MultiAlgo

    Fem parallelle mining-algoritmer for å hindre hashrate-sentralisering.

  4. 2017

    SegWit

    Protokolloppgradering som optimerer kapasitet og løser transaksjons-malleabilitet.

  5. 2019

    Odocrypt & Dandelion++

    ASIC-resistent konsensus og personvern-bevarende transaksjonsruting.

  6. 2020+

    DigiAssets & Digi-ID

    Økosystemomfattende integrasjon for on-chain-aktiva og desentralisert autentisering.

  7. 2025

    Core v8.26.2

    Modernisert node-infrastruktur bygget på Bitcoin Core 26-rammeverket.

  8. 2026

    DigiDollar Snart tilgjengelig

    En fullt non-custodial stablecoin som genererer USD-bundet likviditet via tidslåste DGB-reserver.

Fellesskap
