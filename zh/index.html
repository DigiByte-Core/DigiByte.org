为持续运行而设计。

DigiByte 是一条专为规模化设计、采用多算法的高安全 UTXO 区块链。15 秒出块,十二年从未中断运行,为 DGB、DigiAssets、去中心化的 Digi-ID 身份认证,以及即将推出的 DigiDollar 提供经过实战检验的可靠基础。

为什么选 DigiByte

为加密世界下一个十年而设计。

一条纯粹的工作量证明 UTXO 链,自 2014 年起持续推出业界首创 — DigiShield、MultiAlgo、SegWit、Odocrypt、Dandelion++、DigiAssets 与 Digi-ID。

15s
出块时间

15 秒出块,快速确认。

比比特币快 40 倍,比莱特币快 10 倍。15 秒确认意味着 POS 支付、交易所充值和资产转移都能在你读完这句话的时间内完成 — 没有慢链常见的拥堵或手续费飙升。

5
共识

五种挖矿算法并行运行。

DigiByte 通过五种独立的工作量证明算法 — SHA-256d、Scrypt、Skein、Qubit 和 Odocrypt — 保护区块。每种算法面向不同的挖矿生态 (CPU、GPU、FPGA、ASIC),Odocrypt 每 10 天自我重新随机化,防止 ASIC 垄断。攻击这条链意味着同时攻击五种算法。

21B
供应

硬顶上限,按设计稀缺。

DGB 在数学上被严格限制为 210 亿枚,按透明的每月递减时间表发行。没有开发税、没有治理解锁、没有隐藏分配。可预测的稀缺性是货币资产可信的根基 — 这也是为什么每一字节的 DGB 都承载着可验证、能经得起时间考验的价值。

12 年运行
运营记录

十二年运行,零停机。

自 2014 年 1 月以来,主网每 15 秒生产一个区块,从未中断。没有深度重组、没有治理分叉、没有紧急硬分叉。这种稳定运行,加上全球数千个独立运营的节点,正是关键基础设施所需要的运营履历 — 几乎没有其他链能拿出来。

DigiAssets

在安全的 UTXO 链上发行自定义资产。

在 DigiByte 网络上原生发行自定义数字资产,从代币、稳定币到身份记录。每个资产 15 秒确认,由全球工作量证明共识守护。

选择资产

可替代代币、收藏品、证书、凭证 — 任何可以在 UTXO 基础层铸造的资产。

定义元数据

设置名称、供应量、可分割性以及锚定 IPFS 的元数据,让钱包和浏览器能够原生呈现。

签名 & 广播

支付几 sats 的 DGB 作为手续费,广播交易后,资产立即在网络中生效。

随处发送

Transfer to any DGB address. Rendered natively by asset-aware DigiByte wallets.

Digi-ID

无密码、开源的身份认证。

Digi-ID 用去中心化的密码学签名代替传统密码,无需任何中心化代理即可保护用户身份。

  • 密钥本地存储,绝对隐私。
  • 原生抗钓鱼设计。
  • 数字和实体访问的即插即用集成。
打开 Digi-ID.io 开发库
和 OAuth 或「使用 X 登录」有什么不同?

传统单点登录依赖中心化数据代理。Digi-ID 完全去中心化:用户直接在设备上对唯一的挑战进行签名完成认证,过程中完全没有第三方服务器。

需要加密货币钱包吗?

不需要,持有加密货币是可选的。虽然许多 DigiByte 钱包原生支持 Digi-ID,用户也可以使用专为安全登录设计的独立认证 App。

开发者中心

毫不妥协的 UTXO 开发基础设施。

使用可靠、经过实战检验的 RPC,以及固定且可预测的费用结构。DigiByte 提供在数分钟内部署节点、在真正去中心化网络上构建的基础。

// Send 100 DGB via Core RPC (Node.js, no external deps)
import http from 'node:http';
const body = JSON.stringify({ jsonrpc: '1.0', id: 'web', method: 'sendtoaddress',
  params: ['DXXXXXXXXXXXXXXXXXXXXXXXXXXXX', 100] });

const req = http.request({
  hostname: '127.0.0.1', port: 14022, method: 'POST',
  auth: 'rpcuser:rpcpass',
  headers: { 'Content-Type': 'application/json', 'Content-Length': body.length }
}, res => {
  let d = ''; res.on('data', c => d += c);
  res.on('end', () => console.log(JSON.parse(d)));
});
req.write(body); req.end();
# Listen for new blocks (Python, requests)
import requests, time

URL = "http://rpcuser:rpcpass@127.0.0.1:14022"
last = 0
while True:
    info = requests.post(URL, json={
        "jsonrpc": "1.0", "id": "py", "method": "getblockchaininfo", "params": []
    }).json()["result"]
    if info["blocks"] != last:
        print("new block:", info["blocks"])
        last = info["blocks"]
    time.sleep(5)

为什么在 DGB 上发布?
  • 可预测的费用市场:亚分级别的交易成本确保应用可持续扩张。
  • 免授权交易:原生 UTXO 机制绕开了广泛代币授权所带来的安全风险。
  • 无状态执行:纯 UTXO 环境支持简单、并行、可组合的开发。
运营记录

十余年持续执行。

从早期激活 SegWit 到多算法共识,每一次协议升级都是开源、主网验证、无需中心化资金部署的。

  1. 2014

    创世

    通过公开挖矿公平启动;0.5% 预挖用于赠予和开发。

  2. 2014

    DigiShield

    实时难度调整,确保出块时间稳定。

  3. 2014

    MultiAlgo

    五种并行挖矿算法,防止算力集中。

  4. 2017

    SegWit

    协议升级,优化容量并修复交易延展性。

  5. 2019

    Odocrypt & Dandelion++

    抗 ASIC 的共识与保护隐私的交易路由。

  6. 2020+

    DigiAssets & Digi-ID

    面向链上资产和去中心化认证的生态系统级集成。

  7. 2025

    Core v9.26.4

    基于 Bitcoin Core 26 框架的现代化节点基础设施。

    • BIP340 — Schnorr signatures
    • BIP341 — Pay-to-Taproot (P2TR)
    • BIP342 — Tapscript
  8. 2026

    DigiDollar Live on mainnet

    完全非托管的稳定币,通过时间锁定的 DGB 储备生成与美元挂钩的流动性。
